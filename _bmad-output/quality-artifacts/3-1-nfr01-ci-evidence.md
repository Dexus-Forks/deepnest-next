## Methodological notes (transparency)

1. **Runner image drift.** Baseline ran on `ubuntu-22.04` (pinned). The `playwright.yml` workflow runs on `ubuntu-latest`, which has migrated to `ubuntu-24.04` since the baseline was captured. This drift inflates the noise floor slightly but is shared by all three CI samples (pre-merge / post-merge), so the *delta* is comparable. A re-baseline on the pinned cell would be the apples-to-apples gold standard but is **not blocking**: the +9.41 % delta is far enough inside the ±20 % window that even a worst-case 10 % runner-cell penalty would still pass.
2. **`xvfb-run` flag delta.** Baseline workflow uses `xvfb-run --auto-servernum`; `playwright.yml` uses `xvfb-run` (no flag). Difference is single-millisecond Xvfb display-allocation overhead — negligible at this scale.
3. **Step wrapper overhead.** GitHub Actions adds `##[group]/##[endgroup]` framing around each step. In the baseline, the wall-clock was measured via `date +%s%N` *inside* the step — i.e., after the framing. Here we measured the full step duration from API timestamps. The delta (605 ms wrapper + 24 ms teardown ≈ 629 ms) is included in the 18 323 ms figure; subtracting it yields the 17 694 ms "bash command" comparison row above. Both are within tolerance.
