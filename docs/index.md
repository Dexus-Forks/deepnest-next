# deepnest-next — Documentation Index

> Generated by Winston (BMAD Architect) on 2026-04-25 from a deep brownfield scan. This index is the primary entry point for AI-assisted development on `deepnest-next`. Point any future BMAD PRD/story workflow at this file.

## Project Overview

- **Type**: Monolith — single Electron desktop application (`project_type_id: desktop`)
- **Primary Language**: TypeScript (UI/services) + JavaScript (legacy core engine, workers)
- **Architecture**: Multi-process Electron with hidden background renderer for compute offloading + native NFP add-on (`@deepnest/calculate-nfp`)
- **Domain**: 2D irregular bin-packing for laser cutters / CNC / plotters

## Quick Reference

- **Tech Stack**: Electron 40, TypeScript 5.8 (strict), Ractive.js, axios, Playwright, electron-builder, `@deepnest/calculate-nfp` (native), `@deepnest/svg-preprocessor` (native)
- **Entry Point (main process)**: `main.js`
- **Entry Point (UI renderer)**: `main/index.html` → compiled `build/ui/index.js`
- **Entry Point (background worker)**: `main/background.html` → `main/background.js`
- **Build**: `npm run build` (= `tsc && electron-builder install-app-deps`)
- **Run**: `npm run start`
- **Test**: `npm test` (Playwright E2E)
- **Architecture Pattern**: GA orchestrator in main renderer; greedy NFP placement in hidden background renderer; IPC fan-out via `ipcMain`

## Getting Started (AI Agents)

When you're tasked to work on this codebase:

1. **Read this index first**, then `architecture.md`, then the area-specific doc.
2. **Respect the legacy/typed split**: `main.js`, `main/deepnest.js`, `main/background.js`, `main/svgparser.js`, `main/util/*.js` are **legacy CommonJS** loaded directly by Electron — no transpile, no unit tests. New code goes under `main/ui/` as TypeScript.
3. **Globals are intentional** (`window.DeepNest`, `window.SvgParser`, `window.config`, `window.nest`) — see ADR-005 in `architecture.md`. Don't add more.
4. **IPC has a contract** (`architecture.md` §5). Adding a channel means touching both sides; constants live in `main/ui/types/index.ts`.
5. **Tests are E2E only** (Playwright, Chromium-only). Below the UI there's no test harness — verify with `npm test` and manual run.
6. **Native modules** are out-of-tree (`@deepnest/calculate-nfp`, `@deepnest/svg-preprocessor`). After any Electron upgrade, run `npm run build` to rebuild them.

## Open TODOs from the Scan

- `main.js` carries two `// todo:` markers (background-response / background-progress) inherited from the cmidgley fork — workers aren't cleanly shut down on app close.
- Background worker count is hard-coded to 1 (legacy was 8). Re-enabling parallelism requires re-checking the `background-start` round-robin.
- `CHANGELOG.md` is stale (2023). Current release notes live on GitHub Releases.
- macOS / Linux release binaries are unsigned. Only locally-built Windows installers are signed via `helper_scripts/sign_windows.js`.
- README announces ongoing Rust rewrites of compute-heavy code into the `@deepnest/*` packages — architecture boundaries with this repo will continue to shift.

## When Building New Features

For a brownfield PRD or story, point the BMAD workflow at:

```
docs/index.md
```

For architecture changes, supersede the affected ADR in `architecture.md` rather than adding a new one in a different file.

For UI-only features: read `component-inventory.md` and follow the existing factory-function pattern (`createX(deps)` in `main/ui/services/` or `main/ui/components/`, wired in `main/ui/index.ts`).

For engine/algorithm changes: read the relevant section of `architecture.md` (§3.3) and treat `main/deepnest.js` + `main/background.js` as a connected pair — changes to one frequently require IPC contract edits.
