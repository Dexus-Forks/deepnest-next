# Component Inventory

> Generated by Winston (BMAD Architect) on 2026-04-25. Inventories the renderer-side modules in `main/ui/`. Legacy non-UI code (`main/deepnest.js`, `main/background.js`, `main/svgparser.js`, `main/util/`) is documented in `architecture.md` §3.3 / §3.4 — not duplicated here.

## Composition root

**`main/ui/index.ts`** — bootstraps the renderer when DOM is ready. Order of operations:

1. Resolve Node modules: `electron.ipcRenderer`, `@electron/remote`, `graceful-fs`, `form-data`, `axios`, `path`, `@deepnest/svg-preprocessor`.
2. `Ractive.DEBUG = false`.
3. `initializeServices()` → creates config + preset services, attaches `window.config`, pushes initial config into `DeepNest`.
4. `loadPresetList()` populates the preset dropdown.
5. `initializeComponents()` → instantiates navigation, parts-view, nest-view, sheet-dialog, import service, export service, nesting service.
6. `initialize*()` UI handlers (preset modal, config form, background progress, drag/drop prevention, message close, parts resize, version label, import/export buttons).
7. `loadInitialFiles()` — pulls SVGs already sitting in the nest scratch dir into the workspace.

The composition root is also where every cross-module callback (`updatePartsCallback`, `displayNestFn`, `saveJsonFn`, `attachSortCallback`, `applyZoomCallback`, `resizeCallback`) is wired — keep this file as the single seam between modules.

## Services (`main/ui/services/`)

Services hold no DOM references. They are the boundary against IPC, native modules, and stable stateful logic.

| Service | Created via | Responsibility | Public surface (high level) |
|---|---|---|---|
| `ConfigService` | `createConfigService(ipcRenderer)` | Synchronous facade over IPC-backed `userData/settings.json`. Mirrors electron-settings shape. | `getSync(key?)`, `setSync(key, value)` / `setSync(partial)`, `resetToDefaultsSync()`, `BOOLEAN_CONFIG_KEYS` constant export |
| `PresetService` | `createPresetService(ipcRenderer)` | Named-config CRUD via `load-presets`/`save-preset`/`delete-preset` IPC. | `loadPresets()`, `getPreset(name)`, `savePreset(name, config)`, `deletePreset(name)` |
| `ImportService` | `createImportService({...})` | Open SVG/DXF, optional preprocessor pass, optional remote DXF→SVG conversion, dispatch into `DeepNest.importsvg`. | `showImportDialog()`, `loadNestDirectoryFiles()` |
| `ExportService` | `createExportService({...})` | Export to JSON / SVG / DXF (DXF via remote converter). Honours `exportWithSheetBoundboarders`, `exportWithSheetsSpace*`. | `exportToJson()`, `exportToSvg()`, `exportToDxf()`, `setExportButton(el)` |
| `NestingService` | `createNestingService({...})` | Owns nesting lifecycle: start/stop, `background-*` IPC handlers, on-result invokes `displayNestFn` and optionally `saveJsonFn`. | `bindEventHandlers()`, `setNestRactive(...)` |

### Key dependencies (services)

- `ConfigService` is the only sanctioned read/write path for user settings — never call `ipcRenderer.invoke('read-config'|'write-config')` directly elsewhere.
- `ImportService` and `ExportService` both use `axios` + `form-data` for converter round-trips. The converter URL is sourced from `config.conversionServer` (default `https://converter.deepnest.app/convert`).
- `NestingService` is the only module that talks to the **background** renderer (via main-process forwarding).

## Components (`main/ui/components/`)

Components hold DOM references and (in two cases) Ractive instances.

| Component | Construct via | DOM root(s) | Notes |
|---|---|---|---|
| `NavigationService` | `createNavigationService({ resizeCallback })` | Top-bar nav buttons + dark-mode toggle | Ractive-free. Re-runs `resize()` after tab change. |
| `PartsViewService` | `createPartsViewService({ deepNest, config, resizeCallback })` | `#parts` table + drag-to-resize handle | Ractive-bound. Owns parts list, sort, zoom, unit-label sync. |
| `NestViewService` | `createNestViewService({ deepNest, config })` | Nest result panel | Ractive-bound. Used by `NestingService` via `getDisplayNestCallback()`. Returned ractive becomes `window.nest`. |
| `SheetDialogService` | `createSheetDialogService({ deepNest, config, updatePartsCallback, resizeCallback })` | Add-sheet modal | Pushes new sheets into `DeepNest.parts`; calls `updatePartsCallback` to re-render. |

## Globals (intentional)

Declared in `index.d.ts` (`Window` augmentation) and `main/ui/types/index.ts`:

| Global | Type | Set by |
|---|---|---|
| `window.config` | `ConfigObject` (`UIConfig` + getters) | `initializeServices()` |
| `window.DeepNest` | `DeepNestInstance` | Set by `main/deepnest.js` (legacy) |
| `window.SvgParser` | `SvgParserInstance` | Set by `main/svgparser.js` (legacy) |
| `window.nest` | `RactiveInstance` | `initializeComponents()` after `nestViewService.getRactive()` |
| `window.loginWindow` | `Window \| null` | Set to `null` at end of `initialize()`; populated by OAuth flow |

Renderer code reaches the legacy globals via the typed `getDeepNest()` / `getSvgParser()` helpers in `main/ui/index.ts` (ADR-005).

## Type surfaces

`main/ui/types/index.ts` re-exports the canonical core types and adds:

- `UIConfig extends DeepNestConfig` — adds `access_token`, `id_token`, `useSvgPreProcessor`, `useQuantityFromFileName`, `exportWithSheetBoundboarders`, `exportWithSheetsSpace`, `exportWithSheetsSpaceValue`.
- `DEFAULT_CONVERSION_SERVER` constant.
- `SvgPanZoomInstance` interface (svg-pan-zoom controller surface).
- `ImportedFile` interface (workspace import row).
- `ConfigObject` type (extends `UIConfig` + sync methods).
- `NestingProgress`, `SelectableNestingResult`, plus the `IPC_CHANNELS` constant set.
- Ractive view-data types: `NestViewData`, `PartsViewData` (used by `RactiveInstance<T>`).

## Static UI assets

- **Icons** (`main/img/`): 30 SVG icons (post-Story-1.1 / DEE-55; was 35 SVG + 1 PNG). Light/dark variants present for `account`, `close`, `shop`, `spin`, `credits`, `unlimited`. New icons should follow the same convention.
- **Fonts** (`main/font/`): LatoLatinWeb (Bold + Regular merged) and LatoLatinWebLight in `woff` + `woff2` only (post-Story-1.1 / DEE-55; the dead-weight `.eot` / `.ttf` / `BoldItalic` / `lato-{hai,lig}-*` demo specimen kit was removed for a measured 1.85 MB installer-size saving).

## Component dependency graph (renderer)

```
                         index.ts (composition root)
                         /        |        |        \
              configService  presetService  …        navigationService
                  │                                       │
                  ▼                                       ▼
       partsViewService ──── sheetDialogService           (no Ractive)
                  │
                  ▼
       importService / exportService / nestingService
                  │             │            │
                  ▼             ▼            ▼
              window.DeepNest  window.SvgParser  background-* IPC
```

Edges are the actual constructor-injected dependencies — there is no DI container, just plain factory functions. New modules should follow the same pattern: a `createX(deps)` factory that returns a typed object, wired in `index.ts`.
