## 1. Scope

`deepnest-next` is a single-process-tree Electron desktop app. It packages four categories of static assets:

| Category | Location | Files | On-disk size | Packaged into installer? |
|---|---|---:|---:|---|
| App icons (visible renderer) | [`main/img/`](../main/img/) | 30 | ~34 KB | Ō£ģ via electron-builder default file inclusion |
| Web fonts | [`main/font/`](../main/font/) | 7 | ~342 KB | Ō£ģ via electron-builder default file inclusion (live `LatoLatinWeb` + `LatoLatinWebLight`, woff/woff2 only) |
| Test fixtures | [`tests/assets/`](../tests/assets/) | 2 | ~94 KB | ŌØī excluded by `package.json` `build.files: ["!test**"]` |
| Installer / OS app icons | repo root | 2 | ~1.13 MB | Ō£ģ referenced by `package.json` `build.{mac.icon, win.icon, linux.icon}` |

**Total committed asset footprint: ~1.59 MB across 41 files** (post-Story-1.1 / DEE-55, down from ~3.36 MB / 71 files; saving = 1.85 MB across 30 files removed ŌĆö see ┬¦6 measured delta).

There are no audio, video, 3D-model, sprite, texture, or shader assets ŌĆö the renderer is a 2D irregular bin-packing UI with no game/AV surface. There is no localisation asset directory (`i18n/`, `locales/`, `lang/`, `translations/`); the app is English-only.

## 3. Catalogue

### 3.1 UI icons ŌĆö `main/img/`

- **30 files**, all SVG (was 35 + 1 PNG; 5 unused removed in Story 1.1 / DEE-55 ŌĆö see ┬¦6).
- **All 30 in active use**, referenced exclusively from [`main/style.css`](../main/style.css). Zero references from `main/index.html`, `main/notification.html`, `main/ui/**/*.ts`, or `main/*.js` ŌĆö CSS is the single point of truth for icon binding.
- **Removed in Story 1.1 (DEE-55)**: `auth0logo.svg`, `background.png`, `logo.svg`, `progress.svg`, `stop.svg` (~32 KB total dead weight). `background.png` was the only PNG; the directory is now SVG-only.
- **Naming convention**: `<name>.svg` is the canonical icon; `<name>_dark.svg` is the variant for dark/coloured chrome (top-nav, `.importsnav .close`); `<name>_light.svg` is the variant for the dark `#nest` view. **`_dark` / `_light` are NOT toggled by `body.dark-mode`** ŌĆö they encode the *background* the icon sits on, fixed by surface.
- **Hottest icon**: `spin.svg` (5 CSS rules, every busy / spinner state).
- **Variant pairs**: `account`, `close`, `shop`, `spin` use `_dark`; `credits`, `unlimited` use `_light`.
- **Asymmetry warning**: New artwork must follow the same suffix convention. Do not introduce `<name>-dark.svg`, `darkmode_<name>.svg`, or any third spelling.

ŌåÆ Authoritative reference: [`docs/deep-dive/g/main__img.md`](./deep-dive/g/main__img.md) ŌĆö full per-file table with CSS line numbers, selector mapping, cleanup-safety notes, and theme-pair rationale.

### 3.2 Web fonts ŌĆö `main/font/`

- **7 files** post-Story-1.1: one live CSS binding file + three live LatoLatin weights (Bold / Regular / Light) ├Ś `{woff, woff2}` (6 files in `main/font/fonts/`).
- **Live binding**: [`main/font/latolatinfonts.css`](../main/font/latolatinfonts.css) declares `LatoLatinWeb` (Bold + Regular merged) and `LatoLatinWebLight` (Light only). Loaded once by `<link>` in [`main/index.html`](../main/index.html).
- **Used by**: `main/style.css:29` (`body { font: ... LatoLatinWeb }`) and `main/style.css:72` (heading rule using `LatoLatinWebLight`).
- **Notification window does NOT use the webfont** ŌĆö `main/notification.html:40` declares `font-family: Arial, sans-serif`. Per project-context ┬¦3, the notification window has no shared CSS with the main app; webfont coverage there would require an explicit `<link>` + `font-family` declaration that does not exist today.
- **Removed in Story 1.1 (DEE-55)** (~1.82 MB total):
  - **P1**: `LatoLatin-BoldItalic.{eot,ttf,woff,woff2}` (~336 KB; had no `@font-face` binding).
  - **P3**: demo specimen kit ŌĆö `lato-hai-webfont.{eot,svg,ttf,woff}`, `lato-lig-webfont.{eot,svg,ttf,woff}`, `lato-{hai,lig}-demo.html`, `stylesheet.css`, `specimen_files/` (recursive), `generator_config.txt` (~828 KB measured; below the original ~1.2 MB estimate).
  - **P4**: `.eot` + `.ttf` for the three live weights ŌĆö `LatoLatin-{Bold,Regular,Light}.{eot,ttf}` (~650 KB). Removed alongside an edit to `latolatinfonts.css` that drops the `.eot` and `.ttf` `src:` URLs (Chromium uses `woff2`/`woff` natively; Electron is Chromium-only).
- **Family-name case sensitivity**: `LatoLatinWeb` ŌēĀ `latolatinweb` in `main/style.css`; the cascade silently falls through to OS fallback if the casing is wrong.
- **`LatoLatinWeb` is a *combined* family** (Bold + Regular share one name with different `font-weight`s); **`LatoLatinWebLight` is a *separate* family** (Light face only). Switching to Light requires changing `font-family`, not `font-weight`. This is a deliberate Font Squirrel pattern ŌĆö do not collapse them into a single name.

## 4. Asset-binding invariants

Each invariant below is a load-bearing constraint for downstream BMad workflows (PRD, Architecture, Story). Violating one will silently regress the build, the renderer, or the test suite.

1. **Icons are bound exclusively from CSS.** No JS/TS file imports an icon URL. Renaming an icon = editing one file (`main/style.css`). Bundlers, electron-builder, and tree-shaking do not need to be aware of icon paths at the JS level.
2. **`spin.svg` is the universal busy indicator.** Reuse it for any new `.spinner` state ŌĆö do not introduce a second spinner glyph; the visual language is intentionally consistent across surfaces.
3. **Dark-mode toggle (`body.dark-mode`) does not swap icon URLs.** It only repaints surrounding chrome / text colour. Dark-mode-specific glyph swaps must be done via CSS filters (`filter: invert(1)`) or by inlining the SVG so `currentColor` flows.
4. **Webfont family names are case-sensitive in `main/style.css`.** `LatoLatinWeb` ŌēĀ `latolatinweb` ŌĆö the cascade silently degrades to the OS fallback on mismatch.
5. **The notification window has no shared CSS.** Adding webfont coverage to `main/notification.html` requires a fresh `<link rel="stylesheet" href="font/latolatinfonts.css">` and an explicit `font-family: "LatoLatinWeb"` declaration; nothing inherits from the main app.
6. **`tests/assets/` count, placement count, and `#importsnav li` count are coupled.** The 54/54 placement assertion is the canary for SVG sub-path decomposition. Drop a fixture, add a fixture, or re-encode a fixture ŌåÆ re-derive both literals from a fresh run.
7. **`tests/assets/` is excluded from the installer** by `!test**` in `package.json` `build.files`. Moving fixtures elsewhere risks shipping them; keep test data under `tests/`.
8. **No icon font.** The earlier deepnest.io build used a Lato-derived icon font; the current build uses individual SVG backgrounds. Reintroducing an icon font would require rewriting every `background-image: url(img/...)` rule in `main/style.css`.

## 5. Packaged-asset surface (electron-builder)

`package.json` `build.files` (per [`docs/deep-dive/h/package.json.md`](./deep-dive/h/package.json.md)) ships:

- `main/**` (includes `main/img/`, `main/font/`, plus the renderer source)
- compiled UI in `build/ui/**`
- top-level entry files (`main.js`, `presets.js`, `notification-service.js`, etc.)
- the two repo-root `icon.{icns,ico}` files (via `build.{mac,win,linux}.icon`)
- explicit exclusions: `!test**`, `!docs/**`, `!_bmad/**`, `!_bmad-output/**`, dev-only configs

This means **every file under `main/img/` and `main/font/` ships in every installer**. Post-Story-1.1 (DEE-55), the dead-weight payload (5 unused icons + the Lato dead-weight kit + `.eot`/`.ttf` legacy fallbacks) has been removed; only the live `LatoLatinWeb` / `LatoLatinWebLight` woff/woff2 set + 30 in-use icons remain.
