## 1. Scope

`deepnest-next` is a single-process-tree Electron desktop app. It packages four categories of static assets:

| Category | Location | Files | On-disk size | Packaged into installer? |
|---|---|---:|---:|---|
| App icons (visible renderer) | [`main/img/`](../main/img/) | 35 | ~66 KB | ✅ via electron-builder default file inclusion |
| Web fonts | [`main/font/`](../main/font/) | 32 | ~2.07 MB | ✅ via electron-builder default file inclusion (subset is dead weight — see §3.2) |
| Test fixtures | [`tests/assets/`](../tests/assets/) | 2 | ~94 KB | ❌ excluded by `package.json` `build.files: ["!test**"]` |
| Installer / OS app icons | repo root | 2 | ~1.13 MB | ✅ referenced by `package.json` `build.{mac.icon, win.icon, linux.icon}` |

**Total committed asset footprint: ~3.36 MB across 71 files.**

There are no audio, video, 3D-model, sprite, texture, or shader assets — the renderer is a 2D irregular bin-packing UI with no game/AV surface. There is no localisation asset directory (`i18n/`, `locales/`, `lang/`, `translations/`); the app is English-only.

## 3. Catalogue

### 3.1 UI icons — `main/img/`

- **35 files**: 34 SVG + 1 PNG.
- **30 in active use**, all referenced exclusively from [`main/style.css`](../main/style.css). Zero references from `main/index.html`, `main/notification.html`, `main/ui/**/*.ts`, or `main/*.js` — CSS is the single point of truth for icon binding.
- **5 unused (cleanup candidates)**: `auth0logo.svg` (1.9 KB), `background.png` (18.8 KB), `logo.svg` (10.5 KB), `progress.svg` (0.7 KB), `stop.svg` (0.7 KB). Total dead weight: ~32 KB.
- **Naming convention**: `<name>.svg` is the canonical icon; `<name>_dark.svg` is the variant for dark/coloured chrome (top-nav, `.importsnav .close`); `<name>_light.svg` is the variant for the dark `#nest` view. **`_dark` / `_light` are NOT toggled by `body.dark-mode`** — they encode the *background* the icon sits on, fixed by surface.
- **Hottest icon**: `spin.svg` (5 CSS rules, every busy / spinner state).
- **Variant pairs**: `account`, `close`, `shop`, `spin` use `_dark`; `credits`, `unlimited` use `_light`.
- **Asymmetry warning**: New artwork must follow the same suffix convention. Do not introduce `<name>-dark.svg`, `darkmode_<name>.svg`, or any third spelling.

→ Authoritative reference: [`docs/deep-dive/g/main__img.md`](./deep-dive/g/main__img.md) — full per-file table with CSS line numbers, selector mapping, cleanup-safety notes, and theme-pair rationale.

### 3.2 Web fonts — `main/font/`

- **32 files** across two CSS binding files, three live LatoLatin weights (Bold / Regular / Light), one dead Italic family, two demo-only families (`latohairline`, `latolight`), four demo HTML specimen pages, generator config, and a `specimen_files/` support directory.
- **Live binding**: [`main/font/latolatinfonts.css`](../main/font/latolatinfonts.css) declares `LatoLatinWeb` (Bold + Regular merged) and `LatoLatinWebLight` (Light only). Loaded once by `<link>` in [`main/index.html`](../main/index.html).
- **Used by**: `main/style.css:29` (`body { font: ... LatoLatinWeb }`) and `main/style.css:72` (heading rule using `LatoLatinWebLight`).
- **Notification window does NOT use the webfont** — `main/notification.html` declares `font-family: Arial, sans-serif`.
- **Cleanup candidates** (descending risk):
  - **Zero risk**: `LatoLatin-BoldItalic.{eot,ttf,woff,woff2}` — no `@font-face` binding (~336 KB).
  - **Low risk** (developer-only specimen pages): `lato-hai-webfont.*`, `lato-lig-webfont.*`, `lato-{hai,lig}-demo.html`, `stylesheet.css`, `specimen_files/`, `generator_config.txt` (~1.2 MB).
  - **Medium risk** (Electron is Chromium-only — `.eot` is IE9, `.ttf` is fallback): drop `.eot` + `.ttf` for the three live weights, keep `.woff2` + `.woff` (~700 KB).
- **Family-name case sensitivity**: `LatoLatinWeb` ≠ `latolatinweb` in `main/style.css`; the cascade silently falls through to OS fallback if the casing is wrong.
- **`LatoLatinWeb` is a *combined* family** (Bold + Regular share one name with different `font-weight`s); **`LatoLatinWebLight` is a *separate* family** (Light face only). Switching to Light requires changing `font-family`, not `font-weight`. This is a deliberate Font Squirrel pattern — do not collapse them into a single name.

→ Authoritative reference: [`docs/deep-dive/g/main__font.md`](./deep-dive/g/main__font.md) — full per-file binding table, loading graph, and cleanup-safety matrix.

### 3.3 Test fixtures — `tests/assets/`

- **2 files**, both SVG glyphs derived from Google Fonts:
  - [`henny-penny.svg`](../tests/assets/henny-penny.svg) — 70 200 bytes, single `<path>`, 36 closed sub-paths.
  - [`mrs-saint-delafield.svg`](../tests/assets/mrs-saint-delafield.svg) — 26 281 bytes, single `<path>`, 44 closed sub-paths.
- **Total combined: 96 481 bytes; 80 closed sub-paths across both files.**
- **Sole consumer**: [`tests/index.spec.ts`](../tests/index.spec.ts) `Upload files` step (lines 120-133). The directory is enumerated dynamically — any `.svg` file dropped here is auto-picked up.
- **Coupling**: the asset count drives two literal assertions in the spec:
  - `#importsnav li` count = `2` (line 132)
  - placement count = `54/54` (lines 185-186)
  Adding / removing / re-encoding any file requires both literals to be re-derived from a fresh test run.
- **Offline-by-design**: both files have the `.svg` extension, so they take the direct import path through [`main/ui/services/import.service.ts`](../main/ui/services/import.service.ts) and never round-trip through the conversion server. Adding a `.dxf` / `.dwg` / `.eps` / `.ps` fixture here would silently turn the test into an online test.
- **Excluded from the packaged installer** by `package.json` `build.files: ["!test**"]`.
- **Implicit follow-ups discovered during DEE-42**: no companion `LICENSE` / `README` records the SIL Open Font License origin of the glyphs (paperwork gap, not a test gap); no checksum or snapshot makes the asset/test coupling explicit.

→ Authoritative reference: [`docs/deep-dive/j/tests__assets.md`](./deep-dive/j/tests__assets.md) — per-file inventory, IPC trace, and extension-points table.

## 4. Asset-binding invariants

Each invariant below is a load-bearing constraint for downstream BMad workflows (PRD, Architecture, Story). Violating one will silently regress the build, the renderer, or the test suite.

1. **Icons are bound exclusively from CSS.** No JS/TS file imports an icon URL. Renaming an icon = editing one file (`main/style.css`). Bundlers, electron-builder, and tree-shaking do not need to be aware of icon paths at the JS level.
2. **`spin.svg` is the universal busy indicator.** Reuse it for any new `.spinner` state — do not introduce a second spinner glyph; the visual language is intentionally consistent across surfaces.
3. **Dark-mode toggle (`body.dark-mode`) does not swap icon URLs.** It only repaints surrounding chrome / text colour. Dark-mode-specific glyph swaps must be done via CSS filters (`filter: invert(1)`) or by inlining the SVG so `currentColor` flows.
4. **Webfont family names are case-sensitive in `main/style.css`.** `LatoLatinWeb` ≠ `latolatinweb` — the cascade silently degrades to the OS fallback on mismatch.
5. **The notification window has no shared CSS.** Adding webfont coverage to `main/notification.html` requires a fresh `<link rel="stylesheet" href="font/latolatinfonts.css">` and an explicit `font-family: "LatoLatinWeb"` declaration; nothing inherits from the main app.
6. **`tests/assets/` count, placement count, and `#importsnav li` count are coupled.** The 54/54 placement assertion is the canary for SVG sub-path decomposition. Drop a fixture, add a fixture, or re-encode a fixture → re-derive both literals from a fresh run.
7. **`tests/assets/` is excluded from the installer** by `!test**` in `package.json` `build.files`. Moving fixtures elsewhere risks shipping them; keep test data under `tests/`.
8. **No icon font.** The earlier deepnest.io build used a Lato-derived icon font; the current build uses individual SVG backgrounds. Reintroducing an icon font would require rewriting every `background-image: url(img/...)` rule in `main/style.css`.

## 5. Packaged-asset surface (electron-builder)

`package.json` `build.files` (per [`docs/deep-dive/h/package.json.md`](./deep-dive/h/package.json.md)) ships:

- `main/**` (includes `main/img/`, `main/font/`, plus the renderer source)
- compiled UI in `build/ui/**`
- top-level entry files (`main.js`, `presets.js`, `notification-service.js`, etc.)
- the two repo-root `icon.{icns,ico}` files (via `build.{mac,win,linux}.icon`)
- explicit exclusions: `!test**`, `!docs/**`, `!_bmad/**`, `!_bmad-output/**`, dev-only configs

This means **every file under `main/img/` and `main/font/` ships in every installer** — including the 5 unused icons (~32 KB) and the demo-only / dead-weight font kit (~1.5 MB combined). A future packaging-size story can address that without touching the live binding surface (`latolatinfonts.css` + `main/style.css` lines 29 / 72).

## 9. Outstanding follow-ups (for downstream BMad chain)

These were surfaced by the DEE-11 per-group deep-dives and re-confirmed during this DP run. They are **not blockers** for the GPC → CP → VP → CA → CE chain that follows DEE-45, but they are candidates for a future cleanup epic:

1. **Asset-cleanup epic** — execute P1+P2+P3 from §6 in three independent commits (~1.57 MB total saving, all zero-or-low risk).
2. **Asset-licence paperwork** — add a `tests/assets/README.md` (and update `LICENSES.md`) recording the SIL Open Font License origin of the two glyph SVGs (DEE-42 invariant #5).
3. **Test-fixture integrity check** — add a checksum / snapshot for `tests/assets/*` to make the asset / `54/54` placement coupling explicit (DEE-42 invariant #7).
4. **Notification-window typography unification** — if a future story wants the notification window to share the main app's typography, it must add an explicit `<link>` to `latolatinfonts.css` and an explicit `font-family` declaration; nothing inherits today.
5. **Re-skin pipeline for `icon.{icns,ico}`** — there is no committed source SVG for the app icon; re-skinning today means producing both formats out-of-tree. Consider committing an `icon-source.svg` and a regeneration script under `helper_scripts/` if a re-skin lands.
