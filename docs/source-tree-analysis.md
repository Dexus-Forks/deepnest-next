## Critical Directories

| Path | Purpose | Owners |
|---|---|---|
| `main.js` (root file) | App lifecycle, IPC routing, window factory | Main process |
| `main/` | All renderer code, engine, assets | Renderer + workers |
| `main/ui/` | Typed, modular UI surface — *the* place to add new front-end features | UI |
| `main/deepnest.js`, `main/background.js`, `main/svgparser.js` | Core algorithm code (legacy JS) — touch carefully, no unit tests | Engine |
| `main/util/` | Vendored & shared utilities — treat as semi-frozen | Engine |
| `main/util/_unused/` | Quarantined legacy code; do not re-import without refactor | — |
| `tests/` | Playwright E2E only | QA |
| `helper_scripts/` | Build/release helpers (Windows signing, Playwright codegen) | Build |
| `_bmad/` & `_bmad-output/` | BMAD method workspace (this docs run lives in `_bmad-output/.../` policy, but per `_bmad/bmm/config.yaml` the user chose `docs/` for project_knowledge) | Process |

## Entry Points

- **Main process**: `main.js` (referenced by `package.json` `"main"` field).
- **Visible renderer**: `main/index.html` — loaded via `mainWindow.loadURL(file://…/main/index.html)`.
- **Background renderer**: `main/background.html` — loaded by `createBackgroundWindows()`.
- **Notification renderer**: `main/notification.html` — created on demand by `createNotificationWindow(notification)`.
- **TypeScript composition root** (renderer side): `main/ui/index.ts` → compiled to `build/ui/index.js`, included by `main/index.html`.

## Where Things Live (Quick Lookups)

- Add a new config field → `index.d.ts` (`DeepNestConfig`) + `main/ui/types/index.ts` (`UIConfig`) + default in `services/config.service.ts` + form binding in `main/index.html` + handle in `initializeConfigForm()` in `main/ui/index.ts`.
- Add a new IPC channel → declare on both sides; for renderer-side constants, extend `IPC_CHANNELS` in `main/ui/types/index.ts`; register handler in `main.js`.
- Add a new export format → extend `services/export.service.ts` + add a button in `main/index.html` + bind in `initializeExportButtons()` (`main/ui/index.ts`).
- Tune the placement algorithm → `main/background.js` (greedy core) and/or `main/deepnest.js` (GA outer loop).
- Add new geometry helpers → prefer `main/util/*.ts` (typed); avoid extending `main/util/*.js` unless porting.

## Build Output

- `./build/` — TypeScript compiler output (preserves directory layout under `main/`).
- `./deepnest-v${version}-win32-${arch}/` — `npm run dist` (electron-packager) output.
- `dist/` — `electron-builder` output when building installers.

`build/`, `dist/`, and `node_modules/` are excluded by `.gitignore` and `electron-builder` `files` rules.
