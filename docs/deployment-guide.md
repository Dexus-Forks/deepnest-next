# Deployment Guide

> Generated by Winston (BMAD Architect) on 2026-04-25. Sources: `package.json` (`build`, `scripts`), `.github/workflows/build.yml`, `.github/workflows/build_release.yml`, `helper_scripts/sign_windows.js`.

## Targets at a Glance

| Target | Trigger | Builder | Output | Signing |
|---|---|---|---|---|
| **Windows zip + dir** | `npm run dist` (local) / `build_release.yml` on `release: published` | `@electron/packager` | `deepnest-${TAG}-win32-x64.zip` | None (unsigned packager output) |
| **Windows signed installer** | `npm run build-dist` / `npm run build-dist-signed` (local) | `electron-builder` (`win` block in `package.json`) | Signed `dir` + `zip` artefacts under `deepnest-v${version}-win32-${arch}/` | `helper_scripts/sign_windows.js` (signtool, Certum cert) |
| **macOS zip** | `build_release.yml` on `release: published` | `@electron/packager` | `deepnest-${TAG}-darwin-{x64\|arm64}.zip` | None |
| **Linux zip** | `build_release.yml` on `release: published` | `@electron/packager` | `deepnest-${TAG}-linux-{x64\|arm64}.zip` | None |
| **CI smoke build** | every push to `main` + every PR | `@electron/packager` via `npm run dist` | discarded (build verification) | None |

## Local Distribution

### Plain packager (cross-platform)

```sh
npm run clean
npm run build      # tsc + native rebuild
npm run dist       # @electron/packager . deepnest-v${version} --overwrite ...
```

`npm run dist` uses `cross-replace` to interpolate `$npm_package_version` and explicitly ignores: `.github`, `.vscode`, `.husky`, prior `deepnest-v*` outputs, `out`, `renovate.json`, `.readthedocs.yaml`, `CHANGELOG.md`, `examples`, `.gitignore`, `playwright-report`, `test.log`.

Result lands in `./deepnest-v${version}-win32-${arch}/` (the path includes `win32-${arch}` even on other OSes — packager picks it up from the platform).

### One-shot full build

```sh
npm run dist-all          # = clean-all && npm install && build && dist
npm run dist-all-signed   # signed Windows installer flow
```

### Signed Windows installer

```sh
npm run build-dist-signed
```

Runs `electron-builder build --win --publish never` with the `build` block from `package.json`:
- `target: ['dir', 'zip']`
- `icon: 'icon.ico'`
- `signAndEditExecutable: true`, `forceCodeSigning: true`
- `signExts: ['.node', '.exe', '.dll', '.sys', '.msi', '.appx', '.appxbundle']`
- `signtoolOptions.sign: './helper_scripts/sign_windows.js'`

`helper_scripts/sign_windows.js`:
- Looks up signtool at `process.env.SIGNTOOL_PATH` or `C:\Program Files (x86)\Windows Kits\10\bin\10.0.22621.0\x64\signtool.exe`.
- Skips files whose extension isn't `.exe`, `.dll`, or `.node` (electron-builder also passes other types — these are explicitly skipped here).
- Hard-codes the SHA-1 thumbprint of the Certum code-signing cert (`e0cdd96f315959b8d333807585c4868f22d4f396`) and a Certum timestamp server (`http://time.certum.pl`).
- Requires the cert to be present in the local Windows certificate store. If it isn't, signing will fail before the build completes.

> The thumbprint, timestamp server, and `SIGNTOOL_PATH` are deployment-environment configuration. Do not hard-code overrides; use the env var.

## Release Pipeline (`.github/workflows/build_release.yml`)

Triggered by:
- `release: published` event
- Manual `workflow_dispatch`

Three parallel jobs, all fetching the latest release via `actions/github-script` so the asset upload URL is known up-front.

### `build-and-release-linux`

- Runners: `ubuntu-24.04`, `ubuntu-24.04-arm`
- Node: `22.x`
- Extra: `apt-get install -yq libboost-dev`
- Architecture: `x64` for `ubuntu-24.04`, `arm64` for `ubuntu-24.04-arm`
- Pack: `npx @electron/packager . deepnest-${TAG} --overwrite` then `zip -r`
- Upload: `actions/upload-release-asset@v1` to the latest release

### `build-and-release-macos`

- Runners: `macos-13` (x64), `macos-latest` (arm64)
- Node: `22.x`
- No code signing or notarisation configured

### `build-and-release-windows`

- Runner: `windows-latest`
- Node: `22.x`
- Architecture: `x64` only
- Pack: `npx @electron/packager` then `7z a`
- **No signing** in CI — signing happens only via the local `npm run build-dist-signed` flow.

> **Asset name shape**: `deepnest-${TAG}-${BUILD_FOR_OS}-${ARCH}.zip`, e.g. `deepnest-v1.5.6-darwin-arm64.zip`.

## What Ships Inside the Bundle

`package.json` `files` (npm pack scope):

```
main.js, main/**/*, index.d.ts, node_modules, package.json, icon.icns, icon.ico
```

`build.files` (electron-builder scope) keeps everything by default but excludes:
- Prior `deepnest-v*` outputs and `out/`
- `playwright-*`, `test*`, `*.ts`
- Common dev cruft: `.editorconfig`, `package-lock.json`, dotfiles, IDE configs, CI YAMLs
- Source maps and tarballs are not explicitly stripped — sweep them yourself if size matters.

## Runtime Footprint Notes

- **User data**: `app.getPath('userData')` holds `settings.json` (config) and `presets.json` (named presets). On Windows: `%AppData%/deepnest`. On macOS: `~/Library/Application Support/deepnest`. On Linux: `~/.config/deepnest`.
- **Scratch dir**: `os.tmpdir()/nest` (override via `SAVE_PLACEMENTS_PATH`). Wiped on `app.before-quit`.
- **NFP cache**: `os.tmpdir()/nfpcache` — purged on quit.
- **Crash reports**: collected by Electron `crashReporter` but **never uploaded** (`uploadToServer: false`).

## Network Surface

- **DXF conversion**: outbound HTTPS POST to `https://converter.deepnest.app/convert` (configurable via `config.conversionServer`). Multipart upload, response is the converted SVG.
- **Notification poll**: `notification-service.js` checks remote endpoint every 30 minutes (initial check after 3-second startup delay).
- **OAuth login**: opens an external browser window; receives `login-success` IPC payload from the redirect handler.
- **No telemetry**.

## Pre-release Checklist

Before publishing a GitHub release that triggers `build_release.yml`:

1. Bump `version` in `package.json` (the workflow uses `release.tag_name` not the package version, but they should match for asset names).
2. Run `npm run dist-all` locally on at least one platform to catch native-deps drift before CI burns a full matrix.
3. Confirm `LICENSES.md` is current if dependencies changed.
4. For signed Windows builds, verify the Certum cert thumbprint hasn't rotated. If it has, update `helper_scripts/sign_windows.js`.

## Known Deployment Gaps

- macOS builds are unsigned and unnotarised — Gatekeeper will block them on first launch. Users must right-click → Open or run `xattr -d com.apple.quarantine` manually.
- Linux ships only as a zip — no AppImage, deb, rpm, or snap configured.
- Windows CI builds aren't signed; only locally produced builds are. There is no GH-Actions secret-based code-signing path.
- No auto-update channel is configured (no `electron-updater` integration in code).
